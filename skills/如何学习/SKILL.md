#### 问题 1：切入点设计

```
针对「{主题X}」这个学习主题，请基于YJango的切入点设计方法论回答：

1. 这个主题的根本问题是什么？（就像YJango用"生命如何在环境中生存"作为学习理论的根本问题一样）
2. 学习者在接触这个主题之前，最可能有哪些错误的前置认知或"技术妥协"式的思维框架？
3. 推荐使用哪种切入策略？
   - 日常困惑切入（从学习者当前的痛点出发）
   - 生存/历史视角切入（建立紧迫性和正当性）
   - 反直觉现象切入（制造认知冲突引发好奇）
   - 任务反思切入（重新审视习以为常的行为）
4. 具体的切入问题或场景是什么？
5. 从切入点到第一个核心概念的桥梁是什么？

请给出具体的、可直接使用的切入设计方案，不要泛泛而谈。
```

#### 问题 2：概念顺序编排

```
针对「{主题X}」，请基于YJango的概念编排方法论回答：

1. 这个主题包含哪些核心概念？请列出完整清单。
2. 这些概念之间的依赖关系是什么？哪些必须先学，哪些可以后学？请画出概念依赖图。
3. 推荐使用哪种编排模式？
   - 具体→抽象（从感官可触的例子出发，逐步提炼规律）
   - 问题→拆解→方案（从一个完整问题出发，拆成子问题，逐一解决）
   - 定义→验证→应用（先给出定义，用实践验证，再展示应用场景）
4. 每个概念的抽象层级定位是什么？（具体现象 / 分类 / 属性与关系 / 规律 / 系统原理）
5. 每一步之间的衔接逻辑是什么？（就像YJango的"发现局限→引入新概念解决局限"的递进方式）

请给出从第一个概念到最后一个概念的完整学习路线图。
```

#### 问题 3：正反例设计

```
针对「{主题X}」中的每个核心概念，请基于YJango的正反例设计方法论回答：

对于每个核心概念，请设计：

1. 概念的核心特征提取：该概念的本质属性有哪些？
2. 3个正例设计：
   - 正例1：强调特征组合A+B
   - 正例2：强调特征组合A+C（不同于正例1，但同样满足所有核心特征）
   - 正例3：跨领域示例（促进迁移）
   正例设计原则：求同以体会共性——让学习者通过对比正例，抽象并体会它们之间的共性
3. 2个反例设计：
   - 反例1：看起来像正例但缺少关键特征A（并明确指出违反了哪个特征）
   - 反例2：边界案例——满足部分特征但不满足全部（帮助确立概念边界）
   反例设计原则：求异以确立边界——明确指出该现象具体违反了哪个特征
4. 特征对比表：展示正例和反例在各特征维度上的差异

请确保正反例设计能帮助学习者建构"判别模型"——即区分任意一个事物是否属于该概念的能力。
```

#### 问题 4：常见障碍预判

```
针对「{主题X}」，请基于YJango的学习障碍诊断框架回答：

1. 学习者在这个主题上最可能遇到的障碍有哪些？请从以下七种典型障碍模式中识别适用的：
   - 验证手段与行为目标混淆（用"能否背诵"来判断"是否学会"）
   - 学习的同名障碍（用同一种方法应对不同类型的学习任务）
   - 记言代学（把"背诵语言材料"当成"学习"）
   - 知识坍缩（下层对象丢失 或 上层抽象丢失）
   - 类别固有观（认为某种分类是事物不可改变的本质）
   - 材料瓶颈与模型瓶颈（重复同类材料 或 盲目自信拒绝新材料）
   - 负迁移（用旧认知框架错误套用到新领域）

2. 对于每种识别出的障碍：
   - 具体表现是什么？（在这个主题下会长什么样？）
   - 诊断信号是什么？（学习者说了什么或做了什么能暴露这个障碍？）
   - 解决策略是什么？（具体的、可操作的干预方案）

3. 从断墨寻径五层面框架排查：
   - 目标层：学习者可能把什么目标搞错了？
   - 对象层：哪些概念最容易被模糊理解？
   - 材料层：什么样的材料最容易误导学习者？
   - 存储层：哪些内容最容易遗忘或混淆？
   - 验证层：哪些"假学会"最容易出现？

请给出完整的障碍清单和对应的解决方案。
```

#### 问题 5：验证标准设计

```
针对「{主题X}」，请基于YJango的学习验证方法论回答：

1. 对于每个核心概念，设计以下四种验证方法：

   a) 新例预测验证：
      - 提供3个学习过程中没见过的新例子
      - 学习者需要判断这些新例子是否属于该概念
      - 预期的正确答案和判断逻辑是什么

   b) 表述转换验证：
      - 学习者能否用自己的语言（而非教材原文）解释这个概念？
      - 能否用日常生活的例子来说明？
      - 能否向一个完全不懂的人讲清楚？

   c) 边界判别验证：
      - 提供3个边界案例（处于概念边界上的模糊案例）
      - 学习者能否正确判断并给出理由？

   d) 应用迁移验证：
      - 设计1个跨领域的应用场景
      - 学习者能否识别出该场景与所学概念的联系？
      - 能否正确应用所学知识解决这个新问题？

2. "学会"的判定标准：
   - 能推测未见的情况（泛化能力）
   - 能用多种方式表述概念（灵活性）
   - 能识别应用的条件与边界（准确性）
   - 能在新问题中找到旧知识的应用（迁移能力）

3. 常见的"假学会"清单：
   - 在这个主题下，哪些情况看起来像学会了但实际没有？
   - 如何区分"真学会"和"假学会"？
```

## 执行注意事项

### 浏览器操作规范

1. **切换笔记本**：第一步在 Notebook A 中完成所有5个问题后，再切换到 Notebook B 完成4个问题
2. **等待回答完成**：每次提问后，等待 NotebookLM 完整生成回答（观察"停止"按钮消失）再提下一个问题
3. **回答提取**：使用 `read_page` 工具的 accessibility tree 模式提取回答文本（NotebookLM 是 SPA，普通文本提取方法可能失败）
4. **保存中间结果**：每个问题的回答保存到工作目录，防止丢失

### 质量检查清单

整合输出前，逐项检查：

- [ ] 切入点是否与目标受众的已有经验相关？
- [ ] 概念顺序是否尊重依赖关系？每一步衔接是否自然？
- [ ] 每个核心概念是否都有3个正例 + 2个反例？

## 术语表

| 术语 | 定义 |
|------|------|
| **判别模型** | 判断任意一个现象或对象是否属于某个特定概念（类别）的规律。输入为{所有对象/任意现象}，输出为{此概念，非此概念}。学习方式靠观察正反例建构，而非死记硬背。 |
| **联结模型** | 探究从'一个概念的外延'到'另一个概念的外延'的映射关系，即'文件夹与文件夹之间的关系'。必须建立在判别模型之上。 |
| **三重世界** | 物质世界（感官接触的外部物理世界）、意义世界（大脑对物质现象归类后产生的概念集合）、语言世界（概念的代理符号及组织规则的系统集合）。三者关系为'映射'与'指代'。 |
| **渐构分析法** | "分析学习方法的方法"，通过七步流程为特定知识和特定学习者设计最适学习方案。 |
| **正例** | 满足概念所有特征的具体现象，多个正例用于"求同以体会共性"。 |
| **反例** | 不满足概念某个特征的具体现象，用于"求异以确立边界"，必须明确指出违反了哪个特征。 |
| **泛化能力** | 将已学知识应用于从未见过的新情况的能力，是"学会"的核心标志。 |
| **记言代学** | 把"记住语言描述"当成"学习"的典型错误，只记符号而不建构概念。 |
| **知识坍缩** | 知识的下上结构被压扁：下层（对象层）丢失导致'悬空的经验'——只记定义无法解决实际问题；上层（共象层）丢失导致零散经验无法举一反三；判联错配导致学习方式与知识类型不匹配。 |
| **类别固有观** | 认为概念和分类是物质世界先天固有的、唯一的、不可改变的真理的错误观念。 |
| **适用判别** | 在应用模型之前，判断当前情况是否在模型的适用范围内。 |
| **元认知** | 对自己认知过程的认知——知道自己知道什么、不知道什么、以及自己是怎样学会的。 |
| **有效实例** | 同时包含前因和后果、且多样化的学习材料，是归纳学习的基础。 |
| **映射** | 从一个空间到另一个空间的对应关系，知识本质上是对现实的映射。 |
| **编码** | 大脑选择记录信息的方式，不同的编码方式决定了不同的理解。 |
